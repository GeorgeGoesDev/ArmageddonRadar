# Armageddon Radar — Phase 3.5: Polish & Fixes

**Date:** 2026-07-16
**Status:** Approved design → ready for implementation plan
**Scope:** A focused polish/fix batch between Phase 3 and Phase 4. This batch only.

## Goal

Three targeted improvements: fix the app-wide modal-scroll bug (text cut off,
unscrollable), redesign the list sort into a webstore-style field+direction
dropdown, and give each asteroid a stable identity colour shown across the
radar, cards, and detail.

## Feature 1 — Fix modal scroll (bug)

**Cause:** Every bottom-sheet modal nests a `ScrollView` inside a container with
`maxHeight: '90%'`/`'92%'` but the `ScrollView` has no `flex`/`flexShrink`, so RN
lays it out at full content height; when content exceeds the sheet's max height,
the sheet clips it (`overflow` / the fixed max) instead of scrolling. Affects
long content everywhere (APOD explanation, long asteroid detail, settings, risk
detail).

**Fix:** In each affected sheet, give the `ScrollView` `style={{ flexShrink: 1 }}`
(so it can shrink below its content size within the max-height-bounded parent and
become scrollable). No logic change. Affected files:
`src/screens/ApodSheet.tsx`, `DetailSheet.tsx`, `SettingsSheet.tsx`,
`ImpactRiskSheet.tsx`, `SentryDetailSheet.tsx`, `ImpactReportSheet.tsx`.
(`FilterSheet.tsx` has no ScrollView / short content — unchanged.)

**Verify:** on-device, the APOD explanation and a long asteroid detail scroll
fully to the bottom.

## Feature 2 — Webstore-style sort

Replace the cycle-button (rotates closest→largest→fastest with baked-in
directions) with a **Sort dropdown** exposing field **and** direction.

### Model (`src/utils/listControls.ts`)
- Replace `sort: SortKey` with `sortField: SortField` + `sortDir: SortDir`:
  - `type SortField = 'distance' | 'size' | 'speed' | 'name'`
  - `type SortDir = 'asc' | 'desc'`
- `DEFAULT_CONTROLS`: `sortField: 'distance', sortDir: 'asc'` (closest first — same
  default as today).
- `applyListControls` sorts by the field's key with direction:
  - distance → `missLunar`, size → `diameterAvgM`, speed → `velocityKph`,
    name → `displayName` (`localeCompare`); `asc`/`desc` flips the comparator.
- Export a curated `SORT_OPTIONS: { field; dir; label }[]`:
  Closest first (distance/asc), Farthest first (distance/desc), Largest first
  (size/desc), Smallest first (size/asc), Fastest first (speed/desc), Slowest
  first (speed/asc), Name A–Z (name/asc), Name Z–A (name/desc).
- `activeFilterCount` and the search/filter facets are unchanged.

### UI
- `src/components/SortSheet.tsx` — a compact bottom-sheet menu (same modal idiom)
  listing `SORT_OPTIONS`; the active option is highlighted; selecting one sets
  `sortField`+`sortDir` and closes.
- `src/components/ListControlsBar.tsx` — replace the cycle `Pressable` with a
  "Sort: {activeLabel} ▾" button that opens `SortSheet`. The active label is
  derived by matching `controls.sortField`/`sortDir` against `SORT_OPTIONS`. The
  search box and Filters button (hazardous / min-diameter / within-distance)
  stay as-is.

(`DashboardScreen` only holds `controls` state initialised to `DEFAULT_CONTROLS`
and passes it through — no change needed beyond the type flowing through.)

## Feature 3 — Stable per-asteroid colour

Each asteroid gets a **deterministic identity colour** from its id, shown in all
three surfaces so the same object is recognisable.

### Logic (`src/utils/asteroidColor.ts`, pure)
- `asteroidColor(id: string): string` — hash the id to a hue `0–359` and return
  an HSL string with fixed saturation/lightness tuned for the dark theme
  (e.g. `hsl(${hue}, 70%, 62%)`). Deterministic (same id → same colour).
- Unit-tested: determinism, valid `hsl(...)` format, and that a spread of ids
  produces varied hues.

### Where it shows (threat/hazard info is preserved, moved to rings/badges)
- **RadarView blip:** dot **fill** = `asteroidColor(id)`. Hazard/selection move off
  the fill: **hazardous** → a ring (stroke) in `colors.threatOrange`; **selected**
  → a thicker bright ring + the existing pulsing halo. (Today fill encodes
  hazardous/selected; that moves to the ring so identity colour can own the fill.)
- **AsteroidCard:** the leading status dot becomes `asteroidColor(id)`, plus a thin
  left-edge accent bar in the same colour; the existing "HAZARDOUS" badge stays as
  the hazard signal.
- **DetailSheet:** the header name-dot uses `asteroidColor(id)` (instead of the
  threat colour — the hazardous caption + verdict already convey threat).
